### âœ… **What is `HashSet` in Java?**

A `HashSet` is a **collection** that:
- Stores **unique elements only**
- Uses **hashing** internally for **fast operations**
- Does **not maintain insertion order**

It is backed by a `HashMap` where:
- The element is the **key**
- The value is just a placeholder (like a dummy object)

### ğŸ’¡ How does it prevent duplicates?

When you try to add an element:

1. `HashSet` **calls the `hashCode()`** of the element.
2. Then it checks:
   ğŸ‘‰ **Is any existing element already at that hash code index**?
3. If found, it then calls **`equals()`** to check if itâ€™s truly equal.
4. If equal: it **doesnâ€™t add** the element again.
5. If not: it adds it as a new item (in case of hash collision).

## ğŸ”‘ What is Hashing?

**Hashing** is a technique used to **convert data (like a key)** into a **fixed-size number**
(called a **hash code** or **hash value**) using a **hash function**.
This number is then used as an **index** to store or retrieve data quickly in a data structure
like a **hash table** or **HashMap**.
---
## ğŸ§  Why use Hashing?

- âœ… **Fast data access** â€” Typically **O(1)** time complexity.
- âœ… Great for large datasets.
- âœ… Used in databases, compilers, caches, and cryptography.

---

## ğŸ“¦ How it works (step-by-step):

1. **You provide a key** â€” e.g., `"apple"`.
2. **Hash function** is applied â€” converts `"apple"` into a hash code, say `203`.
3. **Modulo operation** â€” hash code `% size of table` gives you the index:
   `203 % 10 = 3`
4. The value is stored at index 3 in an array.

## âš ï¸ Collisions

Two different keys might get the same hash index. This is called a **collision**.

### ğŸ›  How to handle collisions:
1. **Chaining** â€” Store a list at each index (Java's `HashMap` does this).
2. **Open Addressing** â€” Find another empty slot (used in `HashSet` sometimes).
